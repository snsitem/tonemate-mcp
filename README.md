# 톤메이트 (ToneMate)

카카오톡에서 메시지 보내기 전에 한 번씩 멈칫하는 순간 있잖아요.
"이거 팀장님한테 이렇게 보내도 되나", "화나서 막 썼는데 그냥 보내도 될까",
"답장해야 하는데 뭐라고 하지".

그럴 때 한마디 던지면 상황에 맞게 다듬어주는 MCP입니다.

## 뭘 하나

- **메시지 다듬기** — 보낼 말을 상대(상사·거래처·친구·연인·가족)랑 의도에 맞춰
  정중/간결/사유강조 세 가지로 만들어줘요. 골라서 보내면 됩니다.
- **화난 메시지 식히기** — 욱해서 쓴 문장을 가라앉혀주고,
  "이거 그대로 보내면 이렇게 읽혀요" 하고 미리 귀띔도 해줍니다.
- **답장 도우미** — 받은 메시지에 뭐라 답할지 막막할 때 후보 세 개를 뽑아줘요.
- **맞춤법 점검** — 보내기 전 띄어쓰기, 오타 한 번 봐줍니다.

상대: 상사 / 거래처 / 친구 / 연인 / 가족
의도: 거절 / 사과 / 부탁 / 독촉 / 감사 / 보고 / 일반

## 왜 만들었나

ChatGPT한테 "정중하게 고쳐줘" 하면 매번 결과가 달라요. 어떤 날은 너무 딱딱하고
어떤 날은 너무 풀어지고. 톤메이트는 상대별로 규칙(호칭, 어미, 피해야 할 표현, 톤)을
미리 정해놔서 일관되게 나옵니다. 그리고 한 개만 주는 게 아니라 세 버전을 같이 줘서
상황 봐가며 고를 수 있어요.

## tool

| tool | 하는 일 |
|------|---------|
| polish_message  | 상대·의도에 맞춰 3버전 생성 |
| soften_message  | 화난 메시지 진정 + 뉘앙스 경고 |
| reply_helper    | 받은 메시지 답장 후보 3개 |
| check_korean    | 맞춤법·띄어쓰기 |

## 로컬에서 돌려보기

```bash
pip install -r requirements.txt
python server.py            # HTTP, 기본 8000 포트
TRANSPORT=stdio python server.py   # stdio로 쓰고 싶을 때
```

## 배포

Docker로 빌드해서 올리면 됩니다. 엔드포인트 경로는 `/mcp`.

```bash
docker build -t tonemate .
docker run -p 8000:8000 tonemate
```

PlayMCP in KC에 Git 소스 빌드로 올리는 경우엔 이 저장소 주소랑 main 브랜치,
Dockerfile 경로만 넣으면 알아서 빌드해줍니다.
